# Embedding 与语义检索（Day 2）

> 所属模块：01 · NLP 与深度学习基础 ｜ 学习日期：2026-07-08

## 一句话总结

Embedding 把 token/文本变成稠密向量，让"语义相近"变成"向量相近"；配上相似度度量和 ANN 索引，就是语义检索 / RAG 的地基。

## 我的理解

```
token ID: 1234  （只是编号，1234 与 1235 无语义关系）
   │ Embedding 层 / 模型
   ▼
向量 [0.21,-0.87,...,0.05]  （几百~几千维，"猫"和"狗"向量近，"猫"和"汽车"远）
```

**为什么向量能表示语义**：分布式假设——词义由其上下文决定，常出现在相似上下文的词，训练后向量靠近。语义"分布"在所有维度上，没有哪一维单独等于某概念。

- **one-hot = 编号**：任意两词正交，距离全一样，编码里没语义，还极稀疏。
- **embedding = 坐标**：稠密浮点，坐标间有远近，`king-man+woman≈queen` 这类关系变成向量运算。
- 现代模型（Transformer 类）是**上下文相关**的：同一个词在不同句子里向量不同（"苹果手机" vs "吃苹果"）。

## 核心要点

### 三种相似度度量
- **点积 Dot**：`Σ aᵢbᵢ`，既看方向也看长度；受模长影响。
- **余弦 Cosine**：`a·b/(‖a‖‖b‖)`，只看方向，范围[-1,1]；**语义检索默认选它**，不受文本长短影响。
- **L2 欧氏**：`√Σ(aᵢ-bᵢ)²`，空间直线距离，越小越近（前两个是相似度，越大越近）。
- **关键结论**：向量**归一化后，Cosine / 点积 / L2 三者排序等价**。故常"先归一化再用点积"，又快又等价于 cosine。

### 为什么检索用 embedding
- 关键词检索（BM25）匹配**字面**，换个说法（"宠物医院"vs"动物诊所"）就漏召回。
- embedding 检索匹配**语义**：query 和文档过同一模型得向量，按 cosine 找最近 → 意思相近就能召回，一个字不重合也行。这是 RAG 的检索核心（R）。

### embedding 模型 vs 生成模型的向量
- 生成模型（GPT）内部向量是为"预测下一 token"服务的中间表示，不专为整段语义相似度优化。
- Embedding 模型（BGE / text-embedding-3 / Qwen-Embedding）用**对比学习**专门训练输出语义向量。**做检索用专门 embedding 模型，别用生成模型隐藏层。**

### ANN 索引（HNSW / IVF）—— 检索加速
暴力搜索(Flat)逐个算，O(N)，百万级向量扛不住。ANN = 用**少量精度换大量速度**。
- **IVF**：聚类分桶（nlist 个簇），查询时只搜最近的 `nprobe` 个桶，跳过其余。像"按省份分区，只在口音像的几个省找人"。常配 PQ 压缩省内存。调 `nprobe` 平衡精度/速度。内存友好，适合超大规模。
- **HNSW**：分层图导航。高层稀疏（大跨度快速接近），底层全量（精细逼近），像"全国地图→市区图→街道图"逐层收敛。调 `efSearch` 平衡精度/速度。又快又准、被广泛默认采用，缺点是内存占用大。
- **选型**：内存够、要快准 → HNSW（首选）；向量海量、内存紧、可接受精度略降 → IVF+PQ；几万条以内直接 Flat。

## 踩过的坑 / 易混淆点

- **相似度绝对值别较真**：小模型+短句下"猫vs狗"只有 0.24，看**相对排序**而非绝对分。
- **相似度 vs 距离方向相反**：cosine/点积越大越近，L2 越小越近，别搞反。
- **检索别用生成模型的向量**：目标不同，效果差；用专门 embedding 模型。
- **ANN 是"近似"**：不保证绝对 top-K，`nprobe`/`efSearch` 太小会漏检；调大更准更慢。
- 环境同 Day1：独立 venv 装 `sentence-transformers`，首次跑会下模型（~120MB）；`.venv/` 已 gitignore。

## 面试问答（自测）

- Q: 向量为什么能表示语义？
  A: 分布式假设，相似上下文的词向量靠近；语义分布在所有维度。one-hot 只是编号无语义，embedding 是有远近的坐标。
- Q: Cosine / 点积 / L2 怎么选？
  A: 语义检索默认 cosine（只看方向，不受长度影响）；归一化后三者排序等价，常先归一化再用点积（快）；L2 适合要绝对距离的聚类。
- Q: 为什么检索用 embedding 而非关键词？
  A: 关键词匹配字面、换说法会漏；embedding 匹配语义，意思相近就召回。是 RAG 检索核心。
- Q: HNSW 和 IVF 的原理与区别？
  A: IVF 聚类分桶、只搜最近 nprobe 个桶，内存友好适合超大规模；HNSW 分层图导航（稀疏高层→稠密底层），快且准但内存大。都用搜索宽度参数调精度-速度。
